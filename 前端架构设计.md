# 股票/基金数据系统前端架构设计

## 1. 技术栈选择

### 核心技术栈
- **框架**：React 18
- **语言**：TypeScript 5.0+
- **UI 组件库**：Ant Design 5.0+
- **状态管理**：Redux Toolkit
- **图表库**：ECharts 5.0+（通过 echarts-for-react 集成）
- **路由**：React Router v6
- **HTTP 客户端**：Axios
- **构建工具**：Vite
- **代码质量**：ESLint + Prettier + TypeScript ESLint
- **测试**：Jest + React Testing Library

### 第三方依赖
- **日期处理**：dayjs
- **数据处理**：lodash-es
- **表格组件**：@ant-design/pro-components（高级表格）
- **国际化**：i18next（可选）
- **滚动优化**：react-window（大数据列表）

## 3. 组件设计

### 3.1 布局组件

#### Layout 组件
- **TopNav**：顶部导航栏，包含系统标题、用户信息、全局搜索
- **SideMenu**：侧边菜单，包含功能导航
- **MainContent**：主内容区，包含页面内容和面包屑
- **Footer**：页脚，包含版权信息

### 3.2 数据查询组件

#### SearchBar 组件
- 支持股票/基金代码和名称搜索
- 支持按行业、类型、涨跌幅等筛选
- 支持时间范围选择

#### FilterPanel 组件
- 提供高级筛选选项
- 支持多条件组合筛选
- 筛选条件持久化

### 3.3 数据展示组件

#### DataTable 组件
- 基于 Ant Design Table 或 ProTable
- 支持分页、排序、列自定义
- 支持行内操作

#### DetailCard 组件
- 展示单个股票/基金详细信息
- 支持标签页切换不同信息模块
- 响应式布局

### 3.4 分析图表组件

#### LineChart 组件
- 基于 ECharts 实现的折线图
- 支持多系列数据展示
- 支持时间范围选择和数据缩放

#### CandlestickChart 组件
- 股票K线图实现
- 支持技术指标叠加
- 支持时间周期切换

#### RadarChart 组件
- 多维度指标雷达图
- 支持多实体对比
- 自定义指标权重

### 3.5 建议与风险组件

#### RiskAssessment 组件
- 风险等级可视化展示
- 风险因素分析
- 风险控制建议

#### SuggestionCard 组件
- 基于分析结果的购买建议
- 建议理由说明

#### PortfolioBuilder 组件
- 投资组合构建工具

## 6. API 集成方案

### 6.1 API 服务封装

```typescript
// services/api/client.ts
import axios from 'axios';

const apiClient = axios.create({
  baseURL: 'http://localhost:3000/api',
  timeout: 10000,
  headers: {
    'Content-Type': 'application/json',
  },
});

// 请求拦截器
apiClient.interceptors.request.use(
  (config) => {
    // 可以添加认证token等
    return config;
  },
  (error) => {
    return Promise.reject(error);
  }
);

// 响应拦截器
apiClient.interceptors.response.use(
  (response) => {
    return response.data;
  },
  (error) => {
    // 统一错误处理
    return Promise.reject(error);
  }
);

export default apiClient;
```

### 6.2 API 调用封装

```typescript
// services/api/stocks.ts
import apiClient from './client';

export const stocksApi = {
  getStocks: (params?: any) => apiClient.get('/stocks', { params }),
  getStockDetail: (symbol: string) => apiClient.get(`/stocks/${symbol}`),
  getStockHistory: (symbol: string, params?: any) => apiClient.get(`/stocks/${symbol}/history`, { params }),
  getStockAnalysis: (symbol: string) => apiClient.get(`/stocks/${symbol}/analysis`),
};

// services/api/funds.ts
export const fundsApi = {
  getFunds: (params?: any) => apiClient.get('/funds', { params }),
  getFundDetail: (fundCode: string) => apiClient.get(`/funds/${fundCode}`),
  getFundHistory: (fundCode: string, params?: any) => apiClient.get(`/funds/${fundCode}/history`, { params }),
  getFundAnalysis: (fundCode: string) => apiClient.get(`/funds/${fundCode}/analysis`),
};

// services/api/analysis.ts
export const analysisApi = {
  compare: (data: any) => apiClient.post('/analysis/compare', data),
  portfolio: (data: any) => apiClient.post('/analysis/portfolio', data),
  getSuggestions: (params?: any) => apiClient.get('/analysis/suggestions', { params }),
};
```

## 7. 性能优化策略

### 7.1 前端性能优化

- **代码分割**：使用 React.lazy 和 Suspense 实现路由级别的代码分割
- **组件缓存**：使用 React.memo 缓存组件，避免不必要的重渲染
- **状态管理优化**：合理设计 Redux 状态结构，避免过度渲染
- **图片优化**：使用适当的图片格式和大小，支持懒加载
- **网络请求优化**：实现请求缓存、批量请求、防抖节流

### 7.2 数据处理优化

- **大数据列表**：使用 react-window 实现虚拟滚动
- **图表性能**：ECharts 大数据量渲染优化，如数据抽样、增量渲染
- **计算密集型任务**：使用 Web Worker 处理复杂计算
