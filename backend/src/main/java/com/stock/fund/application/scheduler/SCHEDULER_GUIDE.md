# 调度器模块使用指南

## 概述

调度器模块（Scheduler Module）负责管理所有的定时任务，包括数据采集、提醒检查等周期性任务。该模块遵循Spring Boot的标准定时任务实现方式，支持通过配置文件灵活调整定时任务的执行计划。

## 模块结构

```
src/main/java/com/stock/fund/application/scheduler/
├── SchedulerConfig.java          # 定时任务配置类
├── DataCollectionScheduler.java  # 数据采集调度器
└── AlertScheduler.java          # 提醒检查调度器
```

## 配置项说明

所有定时任务的执行计划都可以通过 `application.yml` 文件进行配置：

```yaml
data:
  collection:
    schedule:
      # 股票基本信息采集定时任务 (默认: 每天凌晨2点)
      stock-basic-cron: "0 0 2 * * ?"
      
      # 基金基本信息采集定时任务 (默认: 每天凌晨3点)
      fund-basic-cron: "0 0 3 * * ?"
      
      # 股票实时行情采集定时任务 (默认: 交易时间内每分钟一次)
      stock-quote-cron: "0 */1 9-15 * * MON-FRI"
      
      # 基金实时净值采集定时任务 (默认: 交易时间内每15分钟一次)
      fund-quote-cron: "0 */15 9-15 * * MON-FRI"
      
      # 交易结束后数据采集定时任务 (默认: 每天16点)
      daily-collection-cron: "0 0 16 * * MON-FRI"
      
      # 提醒检查定时任务 (默认: 每分钟)
      alert-check-cron: "0 */1 * * * ?"
```

## 定时任务说明

### 1. 数据采集调度器 (DataCollectionScheduler)

负责定期从数据源采集股票和基金的相关数据：

- **股票基本信息采集**: 每天凌晨2点执行，更新股票基础信息
- **基金基本信息采集**: 每天凌晨3点执行，更新基金基础信息
- **股票实时行情采集**: 交易时间内每分钟执行一次，获取最新股价
- **基金实时净值采集**: 交易时间内每15分钟执行一次，获取最新净值
- **交易结束后数据采集**: 每天16点执行，采集全天数据

### 2. 提醒检查调度器 (AlertScheduler)

负责定期检查用户的提醒设置，当达到预设条件时触发提醒通知。

## 配置示例

### 修改股票实时行情采集频率

如果需要调整股票实时行情的采集频率，可以在 `application.yml` 中设置：

```yaml
data:
  collection:
    schedule:
      # 每30秒采集一次（注意不要太频繁以免触发API限制）
      stock-quote-cron: "0 */30 * * * ?"
```

### 只在工作日执行采集

```yaml
data:
  collection:
    schedule:
      # 只在周一到周五执行
      stock-basic-cron: "0 0 2 * * MON-FRI"
      fund-basic-cron: "0 0 3 * * MON-FRI"
```

### 节假日暂停采集

可以通过配置不同的cron表达式来适应节假日：

```yaml
data:
  collection:
    schedule:
      # 假设节假日不执行，可以根据实际情况调整
      stock-quote-cron: "0 */1 9-15 * * 1-5"  # 周一到周五
```

## 开发规范

### 添加新的定时任务

1. 在 `SchedulerConfig` 类中添加对应的配置属性
2. 在相应的调度器类中使用 `@Scheduled` 注解定义任务
3. 使用配置文件中的属性，支持默认值回退

### Cron表达式格式

Spring Boot使用标准的Unix cron表达式格式：

```
秒 分 时 日 月 星期 年（可选）
```

常用的cron表达式示例：
- `0 0 2 * * ?` - 每天凌晨2点
- `0 */1 9-15 * * MON-FRI` - 工作日9-15点每分钟
- `0 */15 * * * ?` - 每15分钟
- `0 0 0 * * ?` - 每天午夜

## 注意事项

1. **性能考虑**: 避免过于频繁的任务调度，以免影响系统性能
2. **API限制**: 考虑外部API的调用频率限制，合理设置采集间隔
3. **错误处理**: 所有定时任务都应包含适当的异常处理
4. **日志记录**: 重要的执行步骤应记录日志以便追踪
5. **资源管理**: 及时释放数据库连接等系统资源

## 监控和维护

1. 定期检查日志中的定时任务执行情况
2. 根据业务需求调整定时任务的执行频率
3. 监控系统负载，确保定时任务不会影响正常业务处理
4. 在系统维护期间，可通过配置临时禁用某些定时任务
