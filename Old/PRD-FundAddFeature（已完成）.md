# 基金添加功能需求文档 (PRD)

## 1. 功能概述

### 1.1 项目背景

当前基金管理系统中，基金列表页面的"添加基金"功能为 Stub 状态（仅显示提示信息）。后端已实现通过基金代码从新浪 API 获取基金信息并创建采集目标的 API，前端 API 层也已有封装。本功能需要完成前端交互界面的实现。

### 1.2 目标

为基金管理页面实现完整的"添加基金"功能，用户可以通过输入基金代码快速添加基金到系统进行跟踪管理。

### 1.3 范围

- 前端：基金列表页面的"添加基金"弹窗组件
- 后端：已实现，无需开发

---

## 2. 用户故事

| # | 身份 | 场景 | 期望 |
|---|------|------|------|
| US-1 | 投资者 | 我想添加新基金到跟踪列表 | 输入基金代码即可快速添加 |
| US-2 | 投资者 | 添加时想确认基金信息 | 显示基金名称和类型供确认 |
| US-3 | 投资者 | 输入无效代码 | 看到清晰的错误提示 |
| US-4 | 投资者 | 添加已存在的基金 | 提示基金已存在，无需重复添加 |
| US-5 | 投资者 | 添加成功后 | 自动刷新列表显示新基金 |

## 3. 业务流程图

```
用户点击"添加基金"按钮
        │
        ▼
┌───────────────────────┐
│   显示添加基金弹窗     │
│   - 输入框：基金代码   │
│   - 按钮：取消/确认添加 │
└───────────────────────┘
        │
        ▼
   用户输入基金代码
        │
        ▼
   点击"确认添加"
        │
        ├──▶ 代码为空 ──────▶ 显示验证错误："请输入基金代码"
        │
        ├──▶ 代码已存在 ────▶ 显示错误："该基金已在列表中"
        │
        └──▶ 调用后端API
                │
                ├──▶ 成功 ──▶ 关闭弹窗 + 刷新列表 + 成功提示
                │
                └──▶ 失败 ──▶ 显示错误信息
```

## 4. 功能需求清单

### 4.1 前端需求

| # | 功能点 | 优先级 | 描述 |
|---|--------|--------|------|
| FR-1 | 添加基金按钮 | P0 | 点击"添加基金"按钮打开弹窗 |
| FR-2 | 基金代码输入 | P0 | 输入框，支持 6 位基金代码 |
| FR-3 | 输入验证 | P0 | 实时校验：非空、纯数字、6 位长度 |
| FR-4 | 重复添加校验 | P0 | 添加前检查基金是否已存在 |
| FR-5 | 提交处理 | P0 | 调用 `fundsApi.addFundTarget(code)` |
| FR-6 | 成功反馈 | P0 | 成功后关闭弹窗，刷新列表，显示成功消息 |
| FR-7 | 错误处理 | P0 | 显示后端返回的错误信息 |
| FR-8 | 加载状态 | P1 | 提交时显示 loading 状态 |
| FR-9 | 取消功能 | P1 | 点击取消关闭弹窗并重置表单 |

### 4.2 后端需求

> **注意**：后端 API 已实现，无需开发。

| # | 接口 | 描述 |
|---|------|------|
| BR-1 | `POST /api/data-collection-targets/createByCode?code={fundCode}` | 根据基金代码创建采集目标 |

## 5. 非功能需求

### 5.1 验证规则

| 字段 | 规则 | 错误提示 |
|------|------|----------|
| 基金代码 | 必填 | "请输入基金代码" |
| 基金代码 | 6 位数字 | "基金代码应为 6 位数字" |
| 基金代码 | 纯数字 | "基金代码只能包含数字" |

### 5.2 错误处理

| 错误场景 | 处理方式 |
|----------|----------|
| 后端返回错误 | 显示后端错误消息 |
| 网络错误 | 显示"网络错误，请重试" |
| 超时 | 显示"请求超时，请重试" |
| 基金不存在 | 显示"未找到该基金，请确认代码是否正确" |
| 基金已存在 | 显示"该基金已在列表中" |

## 6. UI/UX 需求

### 6.1 弹窗形式

- **组件**：使用 Ant Design Modal
- **宽度**：480px
- **标题**："添加基金"

### 6.2 布局结构

```
┌────────────────────────────────┐
│  添加基金                    X │
├────────────────────────────────┤
│                                │
│  基金代码                      │
│  ┌──────────────────────────┐  │
│  │ 请输入6位基金代码         │  │
│  └──────────────────────────┘  │
│  * 请确保基金代码正确        │
│                                │
│  ┌──────────┐  ┌────────────┐  │
│  │   取消   │  │  确认添加   │  │
│  └──────────┘  └────────────┘  │
│                                │
└────────────────────────────────┘
```

### 6.3 交互状态

| 状态 | 视觉表现 |
|------|----------|
| 输入框获得焦点 | 蓝色边框 |
| 输入框有错误 | 红色边框 + 错误提示 |
| 提交按钮 loading | 按钮显示 spinner，禁用点击 |
| 添加成功 | 弹窗关闭，List 刷新 |

### 6.4 成功/失败反馈

| 场景 | 反馈方式 |
|------|----------|
| 添加成功 | `message.success("基金添加成功")` |
| 添加失败 | `message.error(errorMessage)` |
| 验证失败 | 输入框下方红色提示文字 |
