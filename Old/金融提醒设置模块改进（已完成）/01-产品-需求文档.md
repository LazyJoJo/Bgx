## 1. 业务背景

Bgx 是一个金融数据分析与提醒系统，主要功能包括：
- 股票和基金数据的采集、存储与展示
- 基于价格和涨跌幅的价格提醒功能
- 风险提醒自动检测（工作日 11:30 和 14:30 触发）
- 基金分析功能

**提醒设置模块**允许用户创建、管理个性化的价格监控提醒，支持批量操作，并与仪表盘和风险提醒页面联动。

## 3. 用户操作流程

### 3.1 创建提醒流程

```
1. 进入"提醒列表"页面
2. 点击"创建提醒"按钮
3. 选择模式：
   ├─ 快速模式 → 选择标的类型 → 选择标的 → 点击"一键订阅"
   └─ 自定义模式 → 选择标的类型 → 选择标的 → 选择提醒类型 → 设置目标值 → [展开高级选项] → 点击"创建提醒"
4. 查看创建结果弹窗
5. 成功则跳转列表页
```

### 3.2 批量管理流程

```
1. 进入"提醒列表"页面
2. 勾选要操作的提醒（支持全选）
3. 选择操作：批量删除 / 批量启用 / 批量禁用
4. 确认操作
5. 查看操作结果
```

### 3.3 仪表盘联动流程

```
1. 在仪表盘查看活跃提醒数量（可点击）
2. 在仪表盘查看未读消息栏（风险提醒数+价格提醒数）
3. 点击任意卡片跳转到对应列表
```

## 6. 功能移除

> **移除日期**：2026-04-15
> **移除原因**：功能与其他表重复，简化数据模型

### 6.1 移除范围

| 移除项 | 说明 | 替代方案 |
|--------|------|---------|
| price_alert 表 | 用户主动创建的价格监控规则表 | 使用 user_subscription 表 |
| alert_history 表 | price_alert 触发后的历史记录表 | 直接使用 risk_alert 表 |
| price_alert 相关 API | 创建/查询/编辑/删除提醒的接口 | 使用 subscription 相关 API |
| alert_history 相关 API | 查询历史记录的接口 | 不再需要，风险提醒直接查看 risk_alert |

### 6.2 移除后的数据流向

**移除前（原有架构）**：
```
用户创建 price_alert → 定时任务检测 → 满足条件时触发
                                      ↓
                              创建 alert_history（历史记录）
                              创建 risk_alert（风险提醒）
```

**移除后（简化架构）**：
```
user_subscription → 定时任务检测(11:30/14:30) → 满足条件时创建 risk_alert
```

**说明**：
- `user_subscription` 表作为唯一的订阅数据源
- 定时任务检测 `user_subscription` 中激活的订阅
- 满足条件时直接创建 `risk_alert` 记录
- `risk_alert` 表作为唯一的风险提醒记录存储

### 6.3 移除后的功能对比

| 功能 | 移除前 | 移除后 |
|------|--------|--------|
| 订阅管理 | price_alert 表 | user_subscription 表 |
| 风险记录 | alert_history + risk_alert | 仅 risk_alert |
| 定时检测 | 基于 price_alert | 基于 user_subscription |
| 提醒列表 | price_alert 列表 | user_subscription 列表 |
| 历史查看 | alert_history | risk_alert 列表 |

- 风险提醒订阅联动
