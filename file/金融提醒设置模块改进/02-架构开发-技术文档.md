## 1. 技术方案

### 1.1 交互设计

采用**单页切换式**（方案B）：顶部两个大按钮切换快速模式/自定义模式。

**设计原则：**
- 模式切换器置于页面顶部，作为第一个视觉焦点
- 快速模式表单极简，自定义模式表单分区清晰
- 组件间距统一，呼吸感强
- 适当使用图标、色彩点缀

### 1.4 快速模式 vs 自定义模式对比

| 维度 | 快速模式 | 自定义模式 |
|------|---------|-----------|
| 标的数量 | 多选 | 多选 |
| 提醒类型 | 固定涨跌幅 1% | 可选三种 |
| 目标值 | 固定 1%，不可改 | 可自定义 |
| 高级选项 | 无 | 可展开 |
| 适用场景 | 快速订阅 | 精准监控 |

## 3. 数据库设计

### 3.1 字段变更说明

| 操作 | 字段 | 变更内容 |
|------|------|---------|
| **重命名** | `entity_code` → `symbol` | 与Domain实体保持一致 |
| **重命名** | `entity_type` → `symbol_type` | 与Domain实体保持一致 |
| **重命名** | `entity_name` → `symbol_name` | 与Domain实体保持一致 |
| **重命名** | `threshold` → `target_price` | 语义更清晰（存储targetPrice） |

### 3.2 DDD分层字段映射

| Domain实体 | Application DTO | Infrastructure PO | 数据库字段 |
|-----------|---------------|------------------|----------|
| id | id | id | id |
| userId | userId | userId | user_id |
| symbol | symbol | symbol | symbol |
| symbolType | symbolType | symbolType | symbol_type |
| symbolName | symbolName | symbolName | symbol_name |
| alertType | alertType | alertType | alert_type |
| targetPrice | targetPrice | targetPrice | target_price |
| targetChangePercent | targetChangePercent | targetChangePercent | target_change_percent |
| basePrice | basePrice | basePrice | base_price |
| currentValue | currentValue | currentValue | current_value |
| status | status | isActive (转换) | is_active |
| lastTriggered | lastTriggered | lastTriggered | last_triggered |
| description | description | description | description |
| createdAt | createdAt | createdAt | created_at |
| updatedAt | updatedAt | updatedAt | updated_at |
| alertName (可选) | alertName | alertName | alert_name |
| expireDate (可选) | expireDate | expireDate | expire_date |
| remark (可选) | remark | remark | remark |

> **注意**：`status` (String: ACTIVE/TRIGGERED/INACTIVE) 与 `is_active` (Boolean) 的转换在Repository层处理。

### 3.3 price_alert 表（修订后）

```sql
CREATE TABLE IF NOT EXISTS price_alert (
    id BIGSERIAL PRIMARY KEY,
    user_id BIGINT NOT NULL,
    symbol VARCHAR(20) NOT NULL,                    -- 原 entity_code
    symbol_type VARCHAR(20) NOT NULL,               -- 原 entity_type (STOCK/FUND)
    symbol_name VARCHAR(100),                        -- 原 entity_name
    alert_type VARCHAR(50) NOT NULL,                 -- PRICE_ABOVE/PRICE_BELOW/PERCENTAGE_CHANGE
    target_price DECIMAL(10,4),                      -- 原 threshold (存储 targetPrice)
    target_change_percent DECIMAL(10,4),             -- 涨跌幅阈值
    base_price DECIMAL(10,4),                        -- 基准价格
    current_value DECIMAL(10,4),                     -- 当前值
    is_active BOOLEAN DEFAULT TRUE,                  -- 是否激活
    status VARCHAR(20) DEFAULT 'ACTIVE',             -- 状态: ACTIVE/TRIGGERED/INACTIVE
    last_triggered TIMESTAMP WITHOUT TIME ZONE,     -- 最后触发时间
    description TEXT,                                -- 描述
    created_at TIMESTAMP WITHOUT TIME ZONE DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP WITHOUT TIME ZONE DEFAULT CURRENT_TIMESTAMP,
    deleted INTEGER DEFAULT 0,
    -- 可选新增字段
    alert_name VARCHAR(100),                         -- 提醒名称
    expire_date DATE,                                -- 有效期
    remark TEXT                                      -- 备注
);

-- 索引
CREATE INDEX IF NOT EXISTS idx_price_alert_user_symbol ON price_alert(user_id, symbol, symbol_type)
    WHERE deleted = 0 AND is_active = TRUE;
CREATE INDEX IF NOT EXISTS idx_price_alert_active ON price_alert(is_active) WHERE deleted = 0;
CREATE INDEX IF NOT EXISTS idx_price_alert_user_status ON price_alert(user_id, is_active) WHERE deleted = 0;
CREATE INDEX IF NOT EXISTS idx_price_alert_type ON price_alert(alert_type) WHERE deleted = 0;
```

### 3.5 user_subscription 表（新增）

用于存储用户的订阅关系，是风险提醒检测的数据源。

```sql
CREATE TABLE IF NOT EXISTS user_subscription (
    id BIGSERIAL PRIMARY KEY,
    user_id BIGINT NOT NULL,
    symbol VARCHAR(20) NOT NULL,                    -- 标的代码
    symbol_type VARCHAR(20) NOT NULL,               -- 标的类型：STOCK/FUND
    symbol_name VARCHAR(100),                        -- 标的名称
    target_change_percent DECIMAL(10,2) NOT NULL,    -- 目标涨跌幅阈值（如：1.00表示1%）
    status VARCHAR(20) DEFAULT 'ACTIVE',             -- 订阅状态：ACTIVE/INACTIVE
    created_at TIMESTAMP WITHOUT TIME ZONE DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP WITHOUT TIME ZONE DEFAULT CURRENT_TIMESTAMP,
    -- 唯一约束：同一用户同一标的只能有一个活跃订阅
    CONSTRAINT uq_user_subscription UNIQUE (user_id, symbol, symbol_type)
);

-- 索引
CREATE INDEX IF NOT EXISTS idx_user_subscription_user ON user_subscription(user_id);
CREATE INDEX IF NOT EXISTS idx_user_subscription_status ON user_subscription(status) WHERE status = 'ACTIVE';
CREATE INDEX IF NOT EXISTS idx_user_subscription_symbol ON user_subscription(symbol, symbol_type);
```

### 3.6 risk_alert 表（新增）

用于存储每次风险检测的结果记录，与价格提醒独立。

```sql
CREATE TABLE IF NOT EXISTS risk_alert (
    id BIGSERIAL PRIMARY KEY,
    user_id BIGINT NOT NULL,
    symbol VARCHAR(20) NOT NULL,                    -- 标的代码
    symbol_type VARCHAR(20) NOT NULL,               -- 标的类型：STOCK/FUND
    symbol_name VARCHAR(100),                        -- 标的名称
    alert_date DATE NOT NULL,                       -- 风险日期
    time_point VARCHAR(10) NOT NULL,                -- 时间点：11:30 / 14:30
    has_risk BOOLEAN DEFAULT FALSE,                 -- 是否触发风险
    change_percent DECIMAL(10,2),                    -- 涨跌幅（2位小数）
    current_price DECIMAL(10,2),                    -- 当前价格（2位小数）
    yesterday_close DECIMAL(10,2),                   -- 昨日收盘价（2位小数）
    is_read BOOLEAN DEFAULT FALSE,                   -- 是否已读
    triggered_at TIMESTAMP WITHOUT TIME ZONE,       -- 触发时间
    created_at TIMESTAMP WITHOUT TIME ZONE DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP WITHOUT TIME ZONE DEFAULT CURRENT_TIMESTAMP,
    -- 唯一约束：同一用户同一标的同一日期同一时间点只有一条记录
    CONSTRAINT uq_risk_alert UNIQUE (user_id, symbol, alert_date, time_point)
);

-- 索引
CREATE INDEX IF NOT EXISTS idx_risk_alert_user ON risk_alert(user_id);
CREATE INDEX IF NOT EXISTS idx_risk_alert_date ON risk_alert(alert_date);
CREATE INDEX IF NOT EXISTS idx_risk_alert_user_date ON risk_alert(user_id, alert_date);
CREATE INDEX IF NOT EXISTS idx_risk_alert_unread ON risk_alert(user_id, is_read) WHERE is_read = FALSE;
```

## 6. 安全与性能

### 6.1 安全考虑

| 场景 | 措施 |
|------|------|
| 批量操作权限 | 校验用户是否有权限操作目标提醒 |
| 输入校验 | 后端必须校验标的代码格式、目标值范围 |
| SQL注入 | 使用参数化查询 |

### 6.2 性能考虑

| 场景 | 措施 |
|------|------|
| 批量删除大量提醒 | 分批处理，每批 100 条 |
| 批量启用/禁用 | 异步处理 |
| 用户提醒超 1000 条 | 分页 + 懒加载 |

```
RiskAlertList
    └── "批量订阅" 按钮 ──> /subscriptions/batch（创建用户订阅）
```

**说明**：
- 风险提醒的数据来源是 `user_subscription` 表，而非 `price_alert` 表
- 定时任务检测 `user_subscription` 中激活的订阅，判断是否触发风险
- 触发后创建 `risk_alert` 记录
