# Bgx 股票/基金数据系统架构文档

## 1. 项目概述

### 1.1 项目简介

Bgx是一个金融数据分析与提醒系统，主要功能包括：
- 股票和基金数据的采集、存储与展示
- 基于价格和涨跌幅的价格提醒功能
- 风险提醒自动检测（工作日11:30和14:30触发）
- 基金分析功能

### 3.4 API 客户端

```typescript
// services/api/client.ts
class ApiClient {
  private client: AxiosInstance

  constructor() {
    this.client = axios.create({
      baseURL: '/api',
      timeout: 10000,
      headers: { 'Content-Type': 'application/json' },
    })

    // 请求拦截器：添加认证 token
    this.client.interceptors.request.use((config) => {
      const token = localStorage.getItem('auth_token')
      if (token) {
        config.headers.Authorization = `Bearer ${token}`
      }
      return config
    })

    // 响应拦截器：统一错误处理
    this.client.interceptors.response.use(
      (response) => response.data,
      (error) => {
        if (error.response?.status === 401) {
          localStorage.removeItem('auth_token')
          window.location.href = '/login'
        }
        return Promise.reject(error)
      }
    )
  }
}
```

## 8. 关键业务逻辑

### 8.1 风险提醒生成规则

1. **触发时间**：每个工作日的 11:30 和 14:30
2. **监控标的**：用户已订阅的股票/基金
3. **检测条件**：
   - 基于价格提醒中的 `PERCENTAGE_CHANGE` 类型
   - 计算涨跌幅 = |当前价格 - 昨日收盘价| / 昨日收盘价 * 100%
   - 涨跌幅 >= 目标百分比时触发
4. **记录规则**：
   - 每个时间点独立判断，有风险才记录
   - 同一标的同一日期只有一条记录（更新最新）
   - 汇总显示取时间靠后的那条

### 8.2 提醒触发流程

```
1. RiskAlertScheduler 触发 (11:30 或 14:30)
       │
       ▼
2. RiskAlertAppService.checkAndCreateRiskAlerts(timePoint)
       │
       ▼
3. 获取所有激活的价格提醒 (PERCENTAGE_CHANGE 类型)
       │
       ▼
4. 对每个提醒标的获取当前价格
       │
       ▼
5. 计算涨跌幅，判断是否达到阈值
       │
       ▼
6. 达到阈值 → 创建 RiskAlert 记录
       │
       ▼
7. 风险提醒入库，用户可查看
```

## 9. 项目启动

### 9.1 前端启动

```bash
cd app
npm install
npm run dev
# 访问 http://127.0.0.1:8080
```

### 9.2 后端启动

```bash
cd backend
./mvnw spring-boot:run
# 或确保 PostgreSQL 运行后打包运行
./mvnw package
java -jar target/data-collector-1.0-SNAPSHOT.jar
```

### 9.3 数据库要求

- PostgreSQL 数据库
- 执行初始化 SQL 脚本创建表结构
- 配置数据库连接信息
