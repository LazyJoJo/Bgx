## 1.3 功能详细说明

### 1.3.1 编辑基金 Modal

- **位置**: [`app/src/components/funds/FundEditModal.tsx`](app/src/components/funds/FundEditModal.tsx:1)
- **功能**:
  - 接收 fund 对象作为编辑数据
  - 表单预填充当前值（market、category、description、collectionFrequency）
  - 调用 `fundsApi.updateFund` API（`/api/funds/{id}`）
  - 成功后关闭 Modal 并刷新列表
- **字段说明**：
  - `collectionFrequency` 为 **Select** 组件，**必填**
  - `market`、`category`、`description` 为可选字段
  - `dataSource` **不在编辑页面显示**，后端统一默认设置为"新浪财经"
  - `code`、`name`、`type` 为只读展示区域
- **复用**: 布局参考 `FundCreateModal.tsx`

### 1.3.1.1 编辑字段规则

| 字段 | 必填 | 默认值 | 说明 |
|------|------|--------|------|
| `market` | 否 | 保持原值 | 市场（SH/SZ等） |
| `category` | 否 | 保持原值 | 分类标签 |
| `collectionFrequency` | **是** | 15 | 采集频率（分钟），默认15分钟 |
| `dataSource` | 否 | **新浪财经**（后端硬编码） | **编辑页面不显示此字段** |
| `description` | 否 | 保持原值 | 描述信息 |

> **重要**：后端 `FundAppServiceImpl.updateFund()` 中，`dataSource` 字段已被硬编码为 `SINA`（新浪财经），前端编辑页面不显示此字段。

### 1.3.2 删除确认 Dialog

- **位置**: [`app/src/pages/funds/FundList.tsx`](app/src/pages/funds/FundList.tsx:36)
- **功能**:
  - 点击删除按钮弹出 `Modal.confirm()` 确认框
  - 显示基金名称和代码确认信息
  - 确认后调用 `dispatch(deleteFund(id))`
  - 成功后刷新列表
  - 失败显示错误提示

### 1.3.3 Redux Slice

- **位置**: [`app/src/store/slices/fundsSlice.ts`](app/src/store/slices/fundsSlice.ts:52)
- **已实现 thunk**:
  - `updateFund` - 更新基金信息
  - `deleteFund` - 删除基金
  - `activateFund` / `deactivateFund` - 激活/停用基金

## 1.4 编辑功能 - 字段定义

### 1.4.1 不可编辑字段（创建时确定）

| 字段 | 原因 |
|------|------|
| `id` | 主键，系统自动生成，不可变 |
| `code` | 基金代码，创建后不可修改 |
| `type` | 类型（STOCK/FUND），创建时确定，不可修改 |
| `name` | 基金名称，来自外部数据源，不应手动编辑 |
| `createdAt` | 创建时间，系统自动记录 |
| `lastCollectedTime` | 最后采集时间，系统自动更新 |
| `nextCollectionTime` | 下次采集时间，系统自动计算 |

### 1.4.2 可编辑字段

| 字段 | 说明 | 表单类型 | 必填 |
|------|------|----------|------|
| `market` | 市场（SH/SZ等） | Select | 否 |
| `category` | 分类标签 | Select | 否 |
| `description` | 描述信息 | TextArea | 否 |
| `collectionFrequency` | 采集频率（分钟） | Select（数值型） | **是** |
| `dataSource` | ~~数据源配置~~ | ~~Select~~ | **编辑页面不显示，后端默认SINA** |

#### collectionFrequency 选项（数值型，与后端 Integer 对齐）

| value | label |
|-------|-------|
| 1440 | 每日 |
| 60 | 每小时 |
| 15 | 每15分钟 |
| 5 | 每5分钟 |
| 1 | 实时（每分钟） |

### 1.4.3 后端API说明

- 编辑接口 `PUT /api/funds/{id}` 采用**部分更新**策略
- 前端只需提交需要修改的字段
- 后端 `FundAppServiceImpl.updateFund()` 通过 `if (request.getXxx() != null)` 判断，仅更新非 null 字段
- `code`、`name`、`type`、`active` 不在 `UpdateFundRequest` 定义中，无法通过此接口修改

## 1.5 删除功能 - 详细说明

### 1.5.1 确认对话框内容

- **标题**：`确认删除`
- **内容**：`确定要删除基金 "{fundName}" ({fundCode}) 吗？此操作不可撤销。`
- **确认按钮**：`确认删除`（danger 样式）

### 1.5.2 删除前检查

- 当前实现：**直接物理删除**
- 待增强：如果基金有关联的待处理数据采集任务或订阅，应提示用户

### 1.5.3 级联删除（待实现）

> **现状**：当前后端 `FundAppServiceImpl.deleteFund()` 仅执行 `dataCollectionTargetRepository.deleteById(id)`，无级联操作。

建议的删除策略二选一：

**方案A：软删除（推荐，符合 AGENTS.md 规范）**
- 为 `data_collection_target` 表增加 `deleted` 字段（1=已删除，0=正常）
- 删除时设置 `deleted = 1`，保留历史数据关联性
- 查询时自动过滤 `deleted = 0`

**方案B：物理删除 + 级联清理**
- 取消该基金的所有订阅（Subscription）
- 清除相关缓存数据
- 删除或归档历史采集记录（fund_quote 表中该基金的数据）

## 1.12 风险评估

| 风险 | 等级 | 缓解措施 |
|------|------|----------|
| 并发编辑冲突 | 中 | 后端乐观锁，前端操作后刷新 |
| 删除数据不可恢复 | 高 | 确认对话框 + 建议改为软删除（`deleted` 字段） |
| 状态同步延迟 | 低 | 操作后立即刷新列表 |
| 编辑字段遗漏 | 低 | 采用部分更新，仅提交修改字段 |
| 级联删除失败 | 中 | 当前未实现级联删除，建议优先落地软删除 |
