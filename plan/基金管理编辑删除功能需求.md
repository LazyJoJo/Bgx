# 基金管理编辑删除功能需求

## 1.1 概述

| 项目 | 内容 |
|------|------|
| 项目名称 | 基金管理页面编辑和删除功能开发 |
| 需求背景 | 后端已提供完整的CRUD API，但前端仅有桩代码，需要实现完整的编辑和删除功能 |
| 目标用户 | 基金管理人员 |
| 开发周期 | 预计 1-2 周 |

## 1.2 功能清单

| 功能 | 优先级 | 描述 |
|------|--------|------|
| 编辑基金Modal | P0 | 新建 FundEditModal 组件，支持编辑基金信息 |
| 删除基金功能 | P0 | 实现删除确认Dialog，调用后端删除API |
| Redux action完善 | P2 | 添加 updateFund、deleteFund |

## 1.3 功能详细说明

### 1.3.1 编辑基金 Modal

- **位置**: `app/src/components/funds/FundEditModal.tsx`
- **功能**:
  - 接收 fund 对象作为编辑数据
  - 表单预填充当前值（名称、代码等）
  - 调用 updateFund API
  - 成功后关闭 Modal 并刷新列表
- **复用**: 可参考 `FundCreateModal.tsx` 的布局和校验逻辑

### 1.3.2 删除确认 Dialog

- **位置**: 在 `FundList.tsx` 中改造
- **功能**:
  - 点击删除按钮弹出 `Modal.confirm()` 确认框
  - 显示基金名称确认信息
  - 确认后调用 `dispatch(deleteFund(id))`
  - 成功后刷新列表
  - 失败显示错误提示

### 1.3.3 Redux Slice 增强

- **位置**: `app/src/store/slices/fundsSlice.ts`
- **新增 thunk**:
  - `updateFund` - 更新基金信息
  - `deleteFund` - 删除基金

## 1.4 编辑功能 - 字段定义

### 1.4.1 不可编辑字段（创建时确定）

| 字段 | 原因 |
|------|------|
| `id` | 主键，系统自动生成，不可变 |
| `code` | 基金代码，创建后不可修改（后端 updateTarget 方法未更新此字段） |
| `type` | 类型（STOCK/FUND），创建时确定，后端虽允许更新但业务上不应修改 |
| `name` | 基金名称，创建后不可修改（基金名称来自外部数据源，不应手动编辑） |
| `createdAt` | 创建时间，系统自动记录 |
| `lastCollectedTime` | 最后采集时间，系统自动更新 |
| `nextCollectionTime` | 下次采集时间，系统自动计算 |

### 1.4.2 可编辑字段

| 字段 | 说明 | 表单类型 |
|------|------|----------|
| `market` | 市场（SH/SZ/HK等） | Select |
| `category` | 分类标签 | Input |
| `description` | 描述信息 | TextArea |
| `collectionFrequency` | 采集频率（分钟） | InputNumber |

### 1.4.3 后端API限制

- 编辑接口 `PUT /api/data-collection-targets/{id}` 不支持部分更新
- 前端需获取完整对象后提交所有字段
- 后端实际允许修改 type，但业务上 type 应视为不可编辑

## 1.5 删除功能 - 详细说明

### 1.5.1 确认对话框内容

- **标题**：`确认删除基金`
- **内容**：`确定要删除 "{fundName}" 吗？此操作不可恢复。`
- **显示基金代码**：`({fundCode})`

### 1.5.2 删除前检查

- 如果基金有关联的待处理数据采集任务，应提示用户
- 删除后，与该基金相关的订阅信息也需要处理（取消订阅）

### 1.5.3 级联删除

- 取消该基金的所有订阅（Subscription）
- 清除相关缓存数据
- 删除历史采集记录（fund_quote 表中该基金的数据）

## 1.12 风险评估

| 风险 | 等级 | 缓解措施 |
|------|------|----------|
| 并发编辑冲突 | 中 | 后端乐观锁，前端操作后刷新 |
| 删除数据不可恢复 | 高 | 确认对话框，逻辑删除（deleted字段） |
| 状态同步延迟 | 低 | 操作后立即刷新列表 |
| 编辑字段遗漏 | 中 | 前端获取完整对象后提交所有字段 |
| 级联删除失败 | 中 | 删除前检查关联数据，分步处理 |
