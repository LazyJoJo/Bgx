## 一、功能需求规格

### 1.1 价格提醒模块 (Alert)

#### 1.1.1 创建提醒
| 字段 | 类型 | 说明 |
|------|------|------|
| 标的类型 | enum | STOCK(股票) / FUND(基金) |
| 标的 | string | 股票代码或基金代码，支持批量选择 |
| 提醒类型 | enum | PRICE_ABOVE / PRICE_BELOW / PERCENTAGE_CHANGE |
| 目标值 | number | 目标价格或目标涨跌幅百分比 |
| 启用状态 | boolean | true=启用 / false=禁用 |

**业务规则：**
- 同一用户对同一股票/基金只能创建一条提醒
- 如果已存在相同股票/基金的提醒，则跳转到编辑页面而非创建
- 批量选择时，可以批量设置相同的提醒参数
- **批量选择采用可搜索的多选下拉框**，支持按代码或名称搜索，下拉选项显示详细信息（类型、代码、名称、市场等）

#### 1.1.2 提醒列表
| 功能 | 说明 |
|------|------|
| 显示 | 所有已设置的提醒 |
| 操作 | 编辑、删除、启用、禁用 |
| 筛选 | 按标的类型、提醒类型、状态筛选 |
| 搜索 | 按股票/基金代码搜索 |

#### 1.1.3 提醒类型
| 类型 | 说明 | 示例 |
|------|------|------|
| PRICE_ABOVE | 价格超过设定值 | 股价>15.5元 |
| PRICE_BELOW | 价格低于设定值 | 股价<10元 |
| PERCENTAGE_CHANGE | 涨跌幅达到设定百分比 | 涨幅>5% |

### 1.2 风险提醒模块 (Risk Alert)

#### 1.2.1 数据来源
- 有价格提醒设置的股票/基金
- 价格变动超过用户设置的阈值（涨跌幅或具体金额）

#### 1.2.2 产生时间
- 每天 **11:30** 和 **14:30** 各产生一次
- 仅工作日产生（非周末和节假日）

#### 1.2.3 记录规则

**每个时间点独立判断：**
- 有风险 → 记录一条数据
- 无风险 → 不记录数据

**汇总显示规则：**
- 同一股票/基金一天只显示一条汇总数据
- 汇总数据取**时间靠后**的那条（14:30 > 11:30）
- 如果14:30有风险就显示有风险，如果14:30无风险就显示无风险（即使11:30有风险）

**场景分析：**

| 时间 | 11:30 | 14:30 | 汇总显示 | 说明 |
|------|-------|-------|---------|------|
| 场景A | 有风险 | 有风险 | 有风险(14:30) | 显示靠后的 |
| 场景B | 有风险 | 无风险 | **无风险**(14:30) | 14:30无风险，汇总显示无风险 |
| 场景C | 无风险 | 有风险 | 有风险(14:30) | 显示靠后的 |
| 场景D | 无风险 | 无风险 | 无数据 | 都不记录 |

**历史记录：**
- 只记录有风险的时间点
- 11:30有风险 → 记录1条
- 14:30有风险 → 再记录1条
- 列表默认**收缩**，只显示最新1条（时间靠后的）

#### 1.2.4 页面展示
| 功能 | 说明 |
|------|------|
| 日期归类 | 按日期分组显示 |
| 展开/收缩 | 默认收缩，只显示最新(14:30)结果 |
| 展开后 | 显示11:30和14:30两条记录 |

### 1.5 实时数据刷新规则

#### 1.5.1 基金净值数据刷新逻辑

**问题背景：**
- 新浪基金API在非交易时间段返回的数据是上一个交易日的收盘数据
- 但API返回的日期可能仍是今天，需要根据时间判断是否为当日有效数据

**判断规则：**
| 当前时间 | API数据类型 | 应使用的日期 |
|---------|------------|-------------|
| 9:30 ≤ 当前时间 ≤ 15:00 | 当日实时数据 | 当日 |
| 当前时间 < 9:30 | 上一交易日收盘数据 | 昨日 |
| 当前时间 > 15:00 | 上一交易日收盘数据 | 昨日 |
| 周末（周六/周日） | 上一交易日收盘数据 | 昨日 |

**实现逻辑：**
1. 获取当前系统时间和日期
2. 判断当前时间是否在A股交易时间段内（9:30-15:00）
3. 判断当前是否为工作日（周一至周五）
4. 如果不在交易时间段内或为周末，则数据属于上一交易日，使用昨日日期

## 五、用户交互流程

### 5.1 创建提醒流程

```
1. 用户进入"提醒设置"页面
2. 点击"创建提醒"按钮
3. 选择模式：单选模式 / 批量模式
4. 单选模式：直接输入标的代码
5. 批量模式：
   a. 点击多选下拉框展开选项
   b. 输入代码或名称进行搜索（下拉选项实时显示搜索结果）
   c. 下拉选项显示详细信息（类型标签、代码、名称、市场等）
   d. 点击选择多个标的
6. 选择股票/基金后，点击确定
7. 设置提醒类型和目标值
8. 点击保存
9. 系统检查是否已存在该标的的提醒
   - 不存在：创建新提醒
   - 已存在：跳转到编辑页面
10. 返回提醒列表，显示新创建的提醒
```

### 5.2 查看风险提醒流程

```
1. 用户进入"风险提醒"页面
2. 默认显示当天的风险数据（按日期分组）
3. 每组默认收缩，只显示14:30的结果
4. 用户可以点击展开组
5. 展开后显示11:30和14:30两条记录
6. 点击记录可以查看详情
7. 查看后自动标记为已读
```

## 八、附录

### 8.1 术语表

| 术语 | 说明 |
|------|------|
| PriceAlert | 价格提醒，用户设置的价格监控规则 |
| RiskAlert | 风险提醒，系统根据价格提醒产生的高风险数据 |
| 标的 | 用户监控的股票或基金 |
| 涨跌幅 | (当前价格 - 昨日收盘价) / 昨日收盘价 * 100% |
| 已读/未读 | 用户是否查看了风险提醒 |

### 8.2 异常处理

| 场景 | 处理方式 |
|------|----------|
| 批量创建时部分成功 | 返回成功列表和失败列表 |
| 风险数据获取失败 | 使用缓存数据，标记数据可能过期 |
| 非工作日 | 不产生风险数据 |
