## 三、发现的问题与差距

### 🔴 HIGH-1 | DataCollectionScheduler 重复检测风险提醒

**文件：** `backend/src/main/java/com/stock/fund/application/scheduler/DataCollectionScheduler.java`
**位置：** 第 92 行（股票）、第 109 行（基金）

**问题：** `scheduleStockQuoteCollection()` 每分钟执行后调用 `checkAndCreateRiskAlerts()`，`scheduleFundQuoteCollection()` 每 15 分钟也调用一次。`RiskAlertScheduler` 又在 11:30/14:30 独立执行同样的逻辑。**风险提醒被重复检测多次**，且非工作时间也会触发。

**根因：** `DataCollectionScheduler` 中遗留了对 `riskAlertAppService.checkAndCreateRiskAlerts()` 的调用。

**修复：** 删除 `scheduleStockQuoteCollection()` 第 92 行和 `scheduleFundQuoteCollection()` 第 109 行附近的 `checkAndCreateRiskAlerts()` 调用，保留 `RiskAlertScheduler` 作为唯一触发源。

---

### 🔴 HIGH-2 | UserSubscriptionRepositoryImpl 修改输入参数（违反不可变性）

**文件：** `backend/src/main/java/com/stock/fund/infrastructure/repository/subscription/UserSubscriptionRepositoryImpl.java`
**位置：** 第 74-76 行（`saveAll`）、第 101-103 行（`batchInsert`）

**问题：** 循环中执行 `subscriptions.get(i).setId(...)`，直接修改传入的列表对象，违反不可变性原则。

```java
// 问题代码（saveAll 第 74-76 行）
for (int i = 0; i < subscriptions.size(); i++) {
    T subscription = subscriptions.get(i);
    // ...
    subscriptions.get(i).setId(id);  // ❌ 直接修改输入参数
}
```

**修复：** 返回新创建的列表副本，不修改原始输入对象。

---

### 🟡 MEDIUM-1 | getYesterdayClose() 股票返回当日收盘价而非昨日

**文件：** `backend/src/main/java/com/stock/fund/application/service/riskalert/impl/RiskAlertAppServiceImpl.java`
**位置：** 第 223-233 行 `getYesterdayClose()` 方法

**问题：** 对股票类型直接返回 `quote.getClose()`（当日收盘价），不是真正的昨日收盘价，导致涨跌幅计算始终 ≈ 0%。

**修复：** 利用 `stock_quote.change`（涨跌额）反推：`prevClose = close - change`。

---

### 🟡 MEDIUM-2 | targetChangePercent 使用 Double 而非 BigDecimal

**文件：** `backend/src/main/java/com/stock/fund/application/service/subscription/impl/SubscriptionAppServiceImpl.java`
**位置：** 第 55-56 行、第 182 行

**问题：** `targetChangePercent` 从 `BigDecimal` 转为 `Double` 存储，存在精度丢失风险。

**修复：** Java 实体及 DTO 中 `targetChangePercent` 改回 `BigDecimal`；前端 `subscriptions.ts` 类型同步改为 `string`。**数据库无需迁移**（已是 `DECIMAL(10,2)`）。

---

### 🟡 MEDIUM-3 | E2E 测试 AC-5.7 失败（涨跌幅信息断言）

**文件：** `app/tests/e2e/AlertManagement.spec.ts`
**位置：** 第 817 行附近

**问题：** `await expect(firstAlert.locator('text=/%/')).toBeVisible()` 失败。`.ant-collapse-item` 未展开时子内容 `display: none`，导致定位失败；或测试数据中 `latestChangePercent` 为 null 时渲染了 `'-'` 而非 `%`。

**修复：** 先展开 Panel 再断言，或换用更稳健的选择器 `locator('[class*="change"]').first()`。

---

### 🟡 MEDIUM-4 | E2E 测试 AC-5.2 失败（页面只显示提醒列表）

**文件：** `app/tests/e2e/AlertManagement.spec.ts`

**问题：** 测试失败，但从代码看页面确实只显示了提醒列表，无批量订阅/创建按钮。可能是 `waitForListLoaded()` 等待逻辑不够稳定。

**修复：** 检查 `RiskAlertListPage.waitForListLoaded()` 等待逻辑，增加适当重试或显式等待条件。
