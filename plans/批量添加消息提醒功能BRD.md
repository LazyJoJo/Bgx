## 1. 执行摘要

### 1.1 背景

当前系统中，用户只能逐个创建价格提醒，每次需要：
1. 选择标的（股票/基金）
2. 设置提醒类型
3. 输入目标值
4. 提交创建

**痛点分析：**
- 当用户需要监控多只股票/基金时（如行业板块整体监控），操作繁琐且耗时
- 重复设置相同参数的提醒，用户体验差
- 无法快速为同一板块的所有股票设置统一提醒条件

**期望状态：**
- 支持批量选择多个标的
- 一次性设置相同提醒参数
- 自动检测并避免重复提醒
- 清晰展示批量创建结果

### 1.2 业务目标

| 目标 | 成功指标 | 目标值 | 时间线 |
|------|---------|--------|--------|
| 提升提醒设置效率 | 单次批量创建平均耗时 | ≤ 30秒（100个标的以内） | V1.0 |
| 降低操作复杂度 | 用户重复设置提醒率 | ≤ 2% | V1.0 |
| 提高功能可用性 | 批量设置成功率 | ≥ 95% | V1.0 |

## 4. 业务流程

### 4.1 批量添加提醒流程

```yaml
流程名称: 批量添加消息提醒
触发条件: 用户主动进入批量添加页面
参与者: 用户、前端系统、后端服务、消息推送服务

业务流程:
  1. 类型选择 → 用户选择标的类型（股票/基金）
  2. 标的搜索 → 用户输入关键词搜索标的
  3. 标的选择 → 用户勾选多个标的（最多100个）
  4. 参数设置 → 用户设置提醒类型和目标值
  5. 提交验证 → 前端校验表单完整性
  6. 批量创建 → 调用后端批量API
  7. 重复检测 → 后端检测已存在提醒
  8. 结果返回 → 返回成功/失败详情
  9. 结果展示 → 前端展示创建结果
  10. 后续操作 → 用户可重试失败项或返回列表

异常处理:
  - 网络异常: 提示用户重试
  - 部分失败: 展示失败详情，提供重试选项
  - 全部失败: 提示原因，引导用户检查

成功指标:
  - 平均操作时间 < 30秒
  - 批量创建成功率 > 95%
  - 用户满意度 > 4.0/5.0
```

### 4.2 业务流程图（文字版）

```
[用户] → 选择类型(股票/基金)
   ↓
[用户] → 搜索并选择多个标的(≤100)
   ↓
[用户] → 设置提醒类型(价格上限/下限/涨跌幅)
   ↓
[用户] → 输入目标值
   ↓
[系统] → 前端表单验证
   ├─ 验证失败 → 显示错误提示 → 返回修正
   └─ 验证通过 ↓
[系统] → 提交批量创建请求
   ↓
[后端] → 批量处理
   ├─ 检测重复提醒
   ├─ 过滤无效标的
   ├─ 批量插入数据库
   └─ 返回结果(成功列表+失败列表)
   ↓
[前端] → 展示创建结果
   ├─ 全部成功 → 成功提示 → 返回列表
   └─ 部分/全部失败 → 展示详情 → 提供重试选项
```

## 5. 功能需求

### 5.1 核心功能

#### 功能1：类型选择前置
- **描述**：用户必须先选择标的类型（股票/基金），才能进行后续操作
- **业务规则**：
  - 未选择类型时，标的选择框禁用
  - 切换类型时，清空已选标的
- **业务价值**：避免用户混淆不同类型标的，减少操作错误

#### 功能2：标的多选
- **描述**：支持搜索并勾选多个标的
- **业务规则**：
  - 单次最多选择100个标的
  - 支持按代码/名称模糊搜索
  - 支持全选/取消全选当前搜索结果
- **业务价值**：提升批量操作效率

#### 功能3：提醒参数设置
- **描述**：设置统一的提醒类型和目标值
- **业务规则**：
  - 提醒类型：价格上限、价格下限、涨跌幅
  - 价格上限/下限：目标值必须>0
  - 涨跌幅：目标值可正可负
- **业务价值**：统一参数设置，减少重复操作

#### 功能4：批量提交与重复检测
- **描述**：提交批量创建请求，自动检测重复提醒
- **业务规则**：
  - 同一用户对同一标的已存在相同类型提醒时，跳过创建
  - 支持部分成功（部分创建成功，部分因重复跳过）
  - 返回详细结果（成功列表+失败列表+失败原因）
- **业务价值**：避免数据冗余，提升用户体验

#### 功能5：结果展示与失败重试
- **描述**：清晰展示批量创建结果，支持快速重试
- **业务规则**：
  - 显示成功数量和失败数量
  - 失败项显示具体原因
  - 提供"重试失败项"功能
- **业务价值**：透明化操作结果，减少用户困惑

## 6. 业务规则

| 规则ID | 名称 | 类别 | 优先级 | 描述 |
|--------|------|------|--------|------|
| BR-BATCH-001 | 类型前置校验 | 验证 | P0 | 未选择标的类型时，禁止选择标的 |
| BR-BATCH-002 | 标的数量上限 | 验证 | P0 | 单次批量创建最多选择100个标的 |
| BR-BATCH-003 | 目标值格式校验 | 验证 | P0 | 价格必须>0，涨跌幅可正可负 |
| BR-BATCH-004 | 重复提醒检测 | 业务逻辑 | P0 | 同一用户+同一标的+同一提醒类型=重复 |
| BR-BATCH-005 | 部分成功处理 | 业务逻辑 | P0 | 批量创建支持部分成功，不影响已创建的提醒 |
| BR-BATCH-006 | 标的类型一致性 | 验证 | P0 | 批量创建的标的必须是同一类型（股票或基金） |
| BR-BATCH-007 | 切换类型清空 | 交互 | P1 | 切换标的类型时，清空已选标的并提示用户 |
| BR-BATCH-008 | 防重复提交 | 技术 | P1 | 前端防抖+后端幂等校验，防止并发重复提交 |

## 11. 范围界定

### 11.1 In Scope（V1.0包含）

- 类型选择（股票/基金）
- 标的搜索和多选（≤100个）
- 全选/取消全选
- 提醒类型选择（价格上限/下限/涨跌幅）
- 目标值输入和验证
- 批量提交和重复检测
- 结果反馈展示
- 失败重试

### 11.2 Out of Scope（V1.0不包含）

- 混合类型批量（股票和基金混合）
- 差异化参数（为不同标的设置不同参数）
- 定时提醒（指定时间自动创建）
- 提醒联动（多个提醒触发关联动作）
- 批量导入（通过文件导入标的列表）
- 快捷模板（预设常用提醒参数模板）
