## 批量添加消息提醒功能

**文档版本：** V1.0  
**创建日期：** 2026-04-13  
**状态：** 待评审

## 1. 产品概述

### 1.1 产品定位
批量添加消息提醒功能，允许用户一次性为多个股票或基金设置相同类型的价格提醒，提升提醒设置效率。

### 1.2 目标用户
- 需要监控多只股票/基金价格的投资者
- 希望设置统一提醒条件的用户

### 1.3 核心价值
- **效率提升**：从逐个设置改为批量设置，大幅减少操作时间
- **体验优化**：类型选择前置，避免用户混淆股票和基金
- **防重复机制**：自动检测已存在提醒，避免重复创建

### 1.4 成功指标
| 指标 | 目标值 |
|------|--------|
| 批量设置成功率 | ≥ 95% |
| 单次批量添加平均耗时 | ≤ 30秒 |
| 用户重复设置提醒率 | ≤ 2% |

### 2.2 用户故事

#### 故事1：批量添加股票提醒
- **用户故事**：作为投资者，我希望一次性为多只股票设置价格上限提醒，以便在股价达到目标时及时收到通知
- **验收标准**：
  1. Given 用户进入批量添加页面
  2. When 用户选择"股票"类型并选中多只股票
  3. And 设置提醒类型为"价格上限"，目标值为20元
  4. Then 系统为每只股票创建一条价格上限提醒
  5. And 显示创建结果（成功X条，失败Y条）
- **业务价值**：核心功能，解决批量设置效率问题
- **技术风险**：低，API已存在

#### 故事2：类型前置选择
- **用户故事**：作为用户，我希望先选择标的类型（股票/基金）再选择具体标的，以避免混淆不同类型的数据
- **验收标准**：
  1. Given 用户进入批量添加页面
  2. When 用户未选择类型时
  3. Then 多选框处于禁用状态
  4. When 用户选择"股票"类型
  5. Then 多选框启用，且只显示股票选项
  6. When 用户切换到"基金"类型
  7. Then 已选标的清空，多选框只显示基金选项
- **业务价值**：降低用户操作错误率
- **技术风险**：低

#### 故事3：重复提醒检测
- **用户故事**：作为用户，我希望系统自动检测已存在的提醒，避免为同一标的重复创建提醒
- **验收标准**：
  1. Given 用户已为股票A设置了价格上限提醒
  2. When 用户尝试批量添加包含股票A的新提醒
  3. Then 系统提示"股票A已存在价格上限提醒"
  4. And 该标的从批量列表中排除或标记为跳过
  5. And 其他不重复的标的正常创建提醒
- **业务价值**：避免数据冗余，提升用户体验
- **技术风险**：中，需要后端支持部分成功场景

#### 故事4：批量创建结果反馈
- **用户故事**：作为用户，我希望清楚了解批量创建的结果，知道哪些成功、哪些失败
- **验收标准**：
  1. Given 用户提交了批量创建请求
  2. When 请求完成
  3. Then 显示成功列表和失败列表
  4. And 失败项显示具体原因（如：已存在提醒、参数错误等）
  5. And 提供"查看失败项"按钮，方便用户修正后重试
- **业务价值**：透明化操作结果，减少用户困惑
- **技术风险**：低

#### 故事5：表单验证
- **用户故事**：作为用户，我希望在提交前获得即时验证反馈，避免提交无效数据
- **验收标准**：
  1. Given 用户未选择任何标的
  2. When 用户点击"提交"
  3. Then 提示"请至少选择一个标的"
  4. Given 用户设置了提醒类型为"价格上限/下限"但未输入目标值
  5. When 用户点击"提交"
  6. Then 提示"请输入目标价格"
  7. Given 用户输入了负数价格
  8. When 用户点击"提交"
  9. Then 提示"价格必须大于0"
- **业务价值**：减少无效请求，提升用户体验
- **技术风险**：低

### 3.2 交互说明

#### 步骤1：选择标的类型
| 交互元素 | 行为 | 状态变化 |
|---------|------|---------|
| 类型单选按钮 | 点击切换类型 | 选中状态切换 |
| 切换到股票 | 清空已选标的，刷新多选框数据 | 多选框显示股票列表 |
| 切换到基金 | 清空已选标的，刷新多选框数据 | 多选框显示基金列表 |
| 未选择类型 | 多选框禁用，提示"请先选择标的类型" | 表单处于初始状态 |

#### 步骤2：选择标的
| 交互元素 | 行为 | 状态变化 |
|---------|------|---------|
| 搜索框 | 输入关键词，实时搜索 | 下拉列表刷新，显示匹配结果 |
| 搜索关键词 | 支持代码、名称模糊搜索 | 匹配结果高亮关键词 |
| 多选框 | 点击勾选/取消 | 更新已选数量 |
| 全选/取消全选 | 一键操作当前搜索结果 | 更新已选数量 |
| 已选数量显示 | 实时更新 | "已选择 X 只股票/基金" |
| 清空选择 | 清空所有已选标的 | 数量归零，多选框重置 |

#### 步骤3：设置提醒参数
| 交互元素 | 行为 | 状态变化 |
|---------|------|---------|
| 提醒类型下拉框 | 选择提醒类型 | 目标值输入框标签和占位符变化 |
| 价格上限/下限 | 选择后显示"元"单位 | 输入框校验：数字且>0 |
| 涨跌幅 | 选择后显示"%"单位 | 输入框校验：数字，可正可负 |
| 目标值输入框 | 输入数值 | 实时校验格式 |

#### 步骤4：提交
| 交互元素 | 行为 | 状态变化 |
|---------|------|---------|
| 提交按钮 | 触发表单验证和提交 | 验证通过：调用API，显示加载状态 |
| 取消按钮 | 关闭弹窗/返回上一页 | 表单数据清空 |
| 提交中 | 按钮禁用，显示"提交中..." | 防止重复提交 |
| 提交完成 | 显示结果弹窗 | 显示成功/失败详情 |

### 3.3 提交后流程

```
提交请求 → 后端处理 → 返回结果
                ↓
    ┌───────────┴───────────┐
    ↓                       ↓
全部成功               部分/全部失败
    ↓                       ↓
显示成功提示            显示详细结果
    ↓                       ↓
返回列表页            提供"重试失败项"选项
```

## 4. 异常场景定义

### 4.1 异常场景清单

| 编号 | 场景 | 触发条件 | 系统行为 | 用户提示 |
|------|------|---------|---------|---------|
| E1 | 未选择类型尝试选择标的 | 用户直接点击多选框 | 多选框禁用，显示提示 | "请先选择标的类型" |
| E2 | 未选择任何标的 | 用户点击提交 | 阻止提交 | "请至少选择一个标的" |
| E3 | 未设置目标值 | 用户点击提交 | 阻止提交 | "请输入目标值" |
| E4 | 目标值格式错误 | 输入非数字字符 | 输入框标红 | "请输入有效数字" |
| E5 | 价格为负数 | 价格上限/下限输入负数 | 阻止提交 | "价格必须大于0" |
| E6 | 网络请求失败 | API调用超时或失败 | 显示错误状态 | "网络异常，请重试" |
| E7 | 部分标的已存在提醒 | 批量创建时部分重复 | 部分成功，返回详情 | "X条成功，Y条失败（标的已存在提醒）" |
| E8 | 所有标的均已存在提醒 | 批量创建时全部重复 | 全部失败，返回详情 | "所选标的均已存在提醒" |
| E9 | 标的数量超限 | 选择超过100个标的 | 阻止选择 | "单次最多选择100个标的" |
| E10 | 搜索无结果 | 搜索关键词无匹配 | 显示空状态 | "未找到匹配的标的" |
| E11 | 切换类型丢失已选 | 用户切换股票/基金 | 清空已选，提示确认 | "切换类型将清空已选标的，是否继续？" |
| E12 | 并发重复提交 | 用户快速多次点击提交 | 防抖处理，只提交一次 | "请勿重复提交" |

### 4.2 批量创建部分失败处理

#### 场景描述
用户选择10个标的批量创建提醒，其中3个已存在提醒，7个创建成功。

#### 处理策略
```json
{
  "success": true,
  "data": {
    "totalCount": 10,
    "successCount": 7,
    "failureCount": 3,
    "successList": [
      {"symbol": "000001", "symbolName": "平安银行", "alertId": 101},
      {"symbol": "000002", "symbolName": "万科A", "alertId": 102}
    ],
    "failureList": [
      {"symbol": "000003", "symbolName": "深物业A", "reason": "已存在价格上限提醒"},
      {"symbol": "000004", "symbolName": "深发展A", "reason": "已存在价格上限提醒"}
    ]
  }
}
```

#### 前端展示
```
┌─────────────────────────────────────┐
│  批量创建结果                         │
├─────────────────────────────────────┤
│                                     │
│  ✅ 成功：7 条                       │
│  ❌ 失败：3 条                       │
│                                     │
│  ── 失败详情 ──                      │
│  • 深物业A (000003)：已存在价格上限提醒  │
│  • 深发展A (000004)：已存在价格上限提醒  │
│  • 深长城A (000005)：已存在价格上限提醒  │
│                                     │
│  [返回列表]    [编辑失败项]           │
└─────────────────────────────────────┘
```

### 4.3 边界情况

| 边界情况 | 处理策略 |
|---------|---------|
| 标的列表中混入已删除的标的 | 后端过滤，前端显示失败原因"标的不存在" |
| 目标值精度超限（如小数位过多） | 前端限制2位小数，后端二次校验 |
| 涨跌幅超过合理范围（如>1000%） | 前端提示"涨跌幅超出合理范围"，建议确认 |
| 用户在提交过程中切换页面 | 提示"提交进行中，是否确认离开？" |

|------|-------|---------|---------|

| AC-P2 | 搜索响应 | 测量搜索响应时间 | ≤ 500ms |
| AC-P3 | 批量提交 | 测量100个标的提交时间 | ≤ 5秒 |

### 8.3 安全验收标准

| 编号 | 验收项 | 验收方法 | 预期结果 |
|------|-------|---------|---------|
| AC-S1 | 身份验证 | 未登录访问创建页面 | 重定向到登录页 |
| AC-S2 | 权限校验 | 尝试为其他用户创建提醒 | 返回权限错误 |
| AC-S3 | 防重复提交 | 快速多次点击提交 | 只提交一次 |
| AC-S4 | 输入过滤 | 输入XSS Payload | 被过滤或转义 |
